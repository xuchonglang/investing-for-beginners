# Bug Bounty 能证明协议安全吗？

Bug Bounty 是项目方奖励白帽提交漏洞的机制。它能提高发现问题的概率，但不能证明协议绝对安全。

看赏金计划时关注：

1. 赏金金额是否足够吸引安全研究员。
2. 覆盖范围是否包括核心合约、前端、预言机和桥。
3. 是否公开历史漏洞和修复记录。
4. 是否与审计、监控、暂停机制配合。

高赏金是加分项，但不是免死金牌。很多攻击来自经济机制、治理权限、预言机和跨链假设，不一定只是代码漏洞。

新手判断 DeFi 协议时，应把 Bug Bounty 当作风险管理的一环，而不是重仓理由。
